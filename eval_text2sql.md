# Lab 3: Evaluate Text-to-SQL Query Generation

## Objective

In this lab, you will learn how to evaluate GitHub Copilot's ability to generate SQL queries from natural language. You will compare the SQL generated by Copilot with an expected SQL query to determine whether the generated query is correct.

**Estimated Duration:** 20 minutes

---

# Learning Outcomes

By the end of this lab, you will be able to:

- Write a natural language query
- Use GitHub Copilot to generate SQL
- Compare generated SQL with an expected SQL query
- Define simple evaluation criteria
- Determine whether the generated SQL passes or fails

---

# Prerequisites

Before starting, ensure you have:

- Visual Studio Code
- GitHub Copilot enabled
- Basic knowledge of SQL

---

# Sample Database

Assume you have the following table.

**Employees**

| EmployeeID | Name | Department | Salary |
|------------|------|------------|--------|
| 101 | Alice | HR | 55000 |
| 102 | Bob | IT | 70000 |
| 103 | Carol | IT | 82000 |
| 104 | David | Finance | 65000 |

---

# Step 1: Create a SQL File

Create a file named:

```text
employee_queries.sql
```

---

# Step 2: Write the Natural Language Request

Add the following comment in the SQL file.

```sql
-- Generate a SQL query to retrieve the names of all employees who work in the IT department.
```

---

# Step 3: Use GitHub Copilot

1. Place the cursor below the comment.
2. Trigger GitHub Copilot.
3. Accept the suggested SQL query.

For example, Copilot may generate:

```sql
SELECT Name
FROM Employees
WHERE Department = 'IT';
```

---

# Step 4: Define the Expected SQL

The expected SQL query is:

```sql
SELECT Name
FROM Employees
WHERE Department = 'IT';
```

---

# Step 5: Compare the Results

Compare the generated SQL with the expected SQL using the following evaluation criteria.

| Evaluation Criteria | Expected |
|---------------------|----------|
| Uses `SELECT` statement | ✅ |
| Retrieves only `Name` column | ✅ |
| Reads from `Employees` table | ✅ |
| Filters `Department = 'IT'` | ✅ |
| Returns only IT employees | ✅ |

---

# Step 6: Evaluate the Generated SQL

## Example 1 – PASS

**Copilot Output**

```sql
SELECT Name
FROM Employees
WHERE Department = 'IT';
```

### Evaluation

| Check | Result |
|--------|--------|
| SELECT statement | ✅ |
| Correct column | ✅ |
| Correct table | ✅ |
| Correct filter | ✅ |

**Overall Result:** PASS

---

## Example 2 – FAIL

**Copilot Output**

```sql
SELECT *
FROM Employees;
```

### Evaluation

| Check | Result |
|--------|--------|
| SELECT statement | ✅ |
| Correct column | ❌ |
| Correct filter | ❌ |

**Overall Result:** FAIL

---

## Example 3 – FAIL

**Copilot Output**

```sql
SELECT Name
FROM Employees
WHERE Department = 'Finance';
```

### Evaluation

| Check | Result |
|--------|--------|
| SELECT statement | ✅ |
| Correct table | ✅ |
| Correct department | ❌ |

**Overall Result:** FAIL

---

# Step 7: Record Your Evaluation

| Copilot Output | Pass/Fail | Reason |
|----------------|-----------|--------|
| Correct SQL | ✅ Pass | Meets all expected criteria |
| Missing WHERE clause | ❌ Fail | Returns all employees |
| Wrong department | ❌ Fail | Retrieves incorrect records |
| Uses `SELECT *` | ❌ Fail | Returns unnecessary columns |

# Lab Summary

In this lab, you learned how to evaluate SQL generated by GitHub Copilot from a natural language request. By comparing the generated query with an expected SQL statement and checking it against predefined evaluation criteria, you can determine whether the generated SQL is correct. This approach provides a simple way to assess the accuracy of AI-generated SQL before it is used in applications or databases.
